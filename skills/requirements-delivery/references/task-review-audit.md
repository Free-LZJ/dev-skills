# Task、审查与审计

编辑 `开发清单.md`、审查章节、审计章节或过程记录时，读取本文件。

## Task 总览

`开发清单.md` 中优先使用这个总览表：

| Task | 目标 | 涉及模块 | 验收信号 | 设计 | 代码 | 验证 | 联调 | 当前状态 |
|------|------|----------|----------|------|------|------|------|----------|

每个 Task 优先包含：

- 目标
- 实现描述
- 核心代码
- 阶段化状态
- 验证证据
- 过程记录索引

## 阶段状态

使用阶段表替代笼统的 `已完成`：

| 阶段 | 状态 | 证据 |
|------|------|------|
| 设计 | 已完成 / 需调整 / 不涉及 | `系统分析.md#...` |
| 代码 | 已完成 / 待实现 / 需调整 / 不涉及 | 文件或 PR/提交 |
| 验证 | 已完成 / 待验证 / 失败待修复 / 不涉及 | `测试计划.md#执行记录` |
| 联调 | 已完成 / 待联调 / 阻塞 / 不涉及 | 联调记录 |
| 生产验证 | 已完成 / 待验证 / 不涉及 | 生产或运行态验证记录 |

使用具体当前状态：

- 设计已完成，代码待实现
- 代码已完成，待最小验证
- 后端单测已完成，仍需联调
- 需调整 / 待调整
- 仍需联调 / 待业务确认

## 审查边界

按上下文价值拆分审查内容：

| 内容 | 默认位置 | 规则 |
|------|------|------|
| 验收口径、阶段状态、验证证据索引 | `开发清单.md` | 必须保留，供后续实现和联调恢复当前状态 |
| 审查门禁 / 审查检查项 | `过程记录.md` | 记录审查标准、过程和结论；如存在当前阻断，只在 `当前上下文.md` 或开发清单当前状态同步一句 |
| 审查结论和审计过程 | `过程记录.md` | 默认迁出，开发清单只保留过程记录索引 |
| 详细审计过程 | `过程记录.md` | 默认迁出，不参与恢复上下文 |
| 多轮审查记录 | `过程记录.md` | 默认迁出，开发清单只留最新结论和索引 |
| 逃逸缺陷、复盘、责任归因 | `过程记录.md` | 默认迁出；当前影响点同步到 `当前上下文.md` 或审查门禁 |
| 测试命令和执行结果 | `测试计划.md` | 开发清单只引用执行记录，不复制长命令流水 |

## 开发清单精简索引块

`开发清单.md` 中默认使用这个结构：

```markdown
**过程记录索引**

| 类型 | 位置 | 当前处理 |
|------|------|------|
| 审计明细 | `过程记录.md#...` | 开发清单只保留阶段状态、验证证据和过程记录索引 |
```

禁止在 `开发清单.md` 中新增 `审查内容` / `审查结果` 二级块或审查明细表。发现残留时，先迁入 `过程记录.md`，再把开发清单降噪为阶段状态、验证指针和过程记录链接。

## 过程记录审查清单

每个活跃 Task 的审查门禁写入 `过程记录.md`，至少覆盖：

- 实现中是否仍包含与当前分析冲突的旧模式、旧状态或旧类型分支。
- 未变化模式和旧行为是否保持稳定。
- 列表、详情、导出、报表链路是否使用一致规则。
- 隐藏字段、空值字段、兼容字段是否符合已确认设计。
- 是否完成最小编译或定向验证。
- 公开 API 返回类型是否重复包裹平台结构。
- 涉及 SQL 时，渲染 SQL 和参数类型是否安全。

## 过程记录模板

长篇记录写入 `过程记录.md`。

### 变更记录

| 日期 | 变更类型 | 变更内容 | 变更原因 | 涉及模块 |
|------|------|------|------|------|

### Bug 修复记录

| 日期 | Bug 描述 | 根因分析 | 修复方案 | 涉及文件 |
|------|------|------|------|------|

### 审计明细

| 日期 | 审查发现 | 根因/风险 | 处理 | 证据 |
|------|------|------|------|------|

追加过程记录后，活跃文档只同步当前结论、受影响 Task 状态、必要回归用例和过程记录链接。

## 已失效结论

旧设计仍保留在文档中时，使用：

```markdown
> [已失效] 本节仅保留历史原因，不得作为实现依据。
> 当前依据：当前上下文.md#当前有效结论
```

已完成 Task 收到新验收口径时，必须退出 `已完成`，直到新口径实现并验证通过。
