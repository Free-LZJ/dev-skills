# 文档集

当需要新建、迁移或修复 requirements-delivery 工作文档时，读取本文件。

## 默认文件

| 文件 | 用途 | 默认内容 |
|------|------|----------|
| `当前上下文.md` | 权威恢复入口与短上下文基线 | 权威文件索引、当前阶段、当前有效结论、已失效结论、活跃 Task、下一步、必须复验项 |
| `系统分析.md` | 权威设计与分析基线 | 背景目标、现状分析、差异分析、详细实现设计、影响范围、风险与假设 |
| `开发清单.md` | 权威任务与状态基线 | 最小粒度 Task、验收口径、阶段化状态、审查门禁、最新结论、验证证据索引 |
| `测试计划.md` | 权威验证基线 | 验证范围、测试场景、测试数据、预期结果、执行记录、遗留风险 |
| `过程记录.md` | 人工审计与历史流水 | 变更记录、Bug 修复记录、审计明细、复盘记录、历史实现迁移记录 |

只有在用户明确要求单文件，或任务小到不值得拆分时，才使用单文件结构。

## 职责边界

- `当前上下文.md` 负责可恢复状态、当前有效结论、已失效结论、活跃 Task、下一步和读取顺序。
- `系统分析.md` 负责当前设计意图和实现说明。
- `开发清单.md` 负责任务拆分、执行状态、当前审查门禁、最新审查结果和证据链接。
- `测试计划.md` 负责验证范围、用例、预期结果、执行记录和遗留风险。
- `过程记录.md` 负责长篇历史和人工审计材料。

某个文件的结论影响其他文件时，必须先同步下游文件，再继续实现。

## 当前上下文.md

此文件应足够短，方便每次继续任务时先读，通常控制在 100-150 行。

推荐结构：

- 权威文件索引
- 当前阶段
- 活跃 Task
- 当前有效结论
- 已失效结论
- 最近状态变化，最多保留 3-5 条高价值记录
- 下一步
- 必须复验项
- 默认读取顺序

推荐表格：

| 主题 | 当前有效结论 | 依据 | 影响 Task |
|------|------|------|------|

| 旧结论 | 失效原因 | 替代结论 | 禁止回退点 |
|------|------|------|------|

## 系统分析.md

优先使用这些章节：

- 背景与目标
- 当前阅读规则
- 当前有效设计摘要
- 现状实现
- 差异分析
- 实现方案
- 影响范围
- 风险与假设

系统分析只保留当前设计和必要约束。长篇变更历史、Bug 历史、多轮审查流水和追责记录迁入 `过程记录.md`。

旧设计仍需保留时，使用 `[已失效]` 标记，并指向替代结论。

## 过程记录.md

此文件默认追加写入，除非要纠正事实错误。适合记录：

- 变更记录
- Bug 修复记录
- 审计明细
- 多轮审查记录
- 复盘记录
- 历史实现迁移记录

活跃文档应链接到过程记录，而不是复制长表格。

## 路径与命名

优先使用稳定命名：

```text
doc/<工单号>/
  当前上下文.md
  系统分析.md
  开发清单.md
  测试计划.md
  过程记录.md
```

如果现有项目已经使用长文件名，不要随意重命名。新增 `当前上下文.md` 和 `过程记录.md`，并在 `当前上下文.md` 中索引已有文件名。
